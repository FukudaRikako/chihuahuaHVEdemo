# UC-17 詳細 — 家族メンバーが緊急時連絡先（動物病院）にワンタップでアクセスする

## 0. Executive Summary

- 対象提言: REC-01（記録・リマインド・通院前サマリに機能を絞った家族共有型 Web アプリの段階導入）の「緊急時導線」要素。`docs/business-requirement.md` の制約（緊急時に記録優先で受診が遅れる懸念 → 緊急時の動物病院連絡導線を全画面に常設）に直接対応する。
- 解決課題: I3（体調変化の早期検知から受診までの導線確保）、補助的に I4（シニア犬ちょこの急変対応）、I5（ルナの低血糖時の緊急受診）。
- 価値: 緊急症状発生時に「電話番号を探す／住所を再入力する」工数を 0 に近づけ、家族の誰が操作しても最短で動物病院に連絡できる状態を担保する。
- 主要リスク3点と先回り策:
  1. 連絡先未登録のまま緊急時を迎えるリスク → 初期セットアップ（UC-03）で「緊急時連絡先」を必須項目化し未登録時はバナーで常時警告。
  2. 動物病院休診時の二次受診先不在（`docs/business-requirement.md` L200 で要確認事項）→ 連絡先を「主／副」2 件まで登録可能とし副が未登録なら警告。
  3. オフライン・通信不安定環境での導線喪失 → 連絡先情報をクライアントキャッシュし、オフラインでも電話アクション（`tel:` リンク）を起動可能にする。

## 1. 前提とスコープ

- スコープ内: 全画面共通の緊急アクション UI（フローティングボタン等）、登録済み動物病院連絡先（電話番号・住所・診療時間メモ）へのワンタップアクセス、電話発信トリガー（`tel:` リンク）、地図アプリ起動トリガー（住所の外部地図アプリ連携）。
- スコープ外: 動物病院との直接予約・チャット・テレヘルス連携、症状の自動診断・トリアージ判定、家族外（獣医師本人）への通知発信、IoT デバイス連動（自動 SOS 等）、救急車・行政窓口連携。
- 主要アクター: 家族メンバー（一次アクター）。
- システム境界: 内＝犬プロフィール配下の緊急時連絡先データ、全画面共通 UI、ワンタップアクションのハンドラ。外＝端末標準の電話アプリ、端末標準の地図アプリ。

## 2. ユースケース詳細（固定スキーマ）

- **ID**: UC-17
- **名称**: 家族メンバーが緊急時連絡先（動物病院）にワンタップでアクセスする
- **業務エリア or ジャーニー段階**: 緊急対応（体調急変時の受診導線）
- **目的**: 体調急変時に、家族メンバーが現在開いている画面を離れずワンタップで登録済み動物病院に電話発信・場所確認できる状態を担保し、受診開始までの所要時間を最小化する。
- **提言/論点ID**: REC-01
- **一次アクター**: 家族メンバー
- **二次アクター/外部システム**: 端末標準の電話アプリ、端末標準の地図アプリ、（間接受益者として）動物病院の獣医師
- **トリガー**: 家族メンバーが画面上の「緊急時連絡先」ボタン（全画面常設）をタップする、または犬詳細画面の緊急連絡先カードをタップする。
- **前提条件**:
  - 家族メンバーが UC-01 でサインアップ済みであり家族グループ（UC-02）に所属していること。
  - 対象犬の緊急時連絡先（主たる動物病院の電話番号・住所、可能なら診療時間メモ）が UC-03 のプロフィール編集で 1 件以上登録済みであること。未登録の場合は基本フローの代替に分岐する。
  - 端末で `tel:` スキームおよび地図アプリ起動スキームが利用可能であること（一般的なスマートフォン／PC ブラウザを想定）。
- **基本フロー要約**:
  1. 家族メンバーが全画面共通の「緊急時連絡先」ボタンをタップする。
  2. システムが現在閲覧中の犬コンテキスト（犬詳細画面なら当該犬、それ以外は家族グループの代表犬または犬選択）に紐づく緊急時連絡先を取得する。
  3. 連絡先情報（病院名・電話番号・住所・診療時間メモ・主／副区分）を最小ステップ（1 画面以内）で提示する。
  4. 家族メンバーが「電話する」をタップすると `tel:` リンクで端末標準の電話アプリを起動、または「地図で開く」で住所を地図アプリへ引き渡す。
  5. システムは緊急アクションの起動を最小監査ログ（誰が・いつ・どの犬に対し・どのアクション）として記録する。
- **主要代替/例外**:
  - 緊急時連絡先が未登録 → 「至急 UC-03 で登録してください」と促す警告画面に遷移し、登録動線を 1 タップで提供。
  - 主動物病院が休診時間帯にあたる（診療時間メモから判定可能な場合のみ）→ 副連絡先（登録があれば）を優先表示し、両方未登録なら警告と一般的な夜間救急の検討メッセージを表示。
  - オフライン／通信断 → クライアントキャッシュの連絡先で `tel:` 起動を継続。監査ログは復帰時に同期。
- **事後条件**: 家族メンバーが端末の電話アプリまたは地図アプリを起動できた状態となり、緊急アクション起動の監査ログが 1 件追加される。アプリ側のデータ更新（記録の作成・更新）は発生させない（緊急時の操作工数を最小化するため）。
- **主要データI/O**:
  - 入力: 犬プロフィール、緊急時連絡先（電話番号／住所／診療時間メモ／主・副区分）、家族メンバー、家族グループ。
  - 出力: 緊急アクション監査ログ（操作者・対象犬・アクション種別・タイムスタンプ）、端末標準アプリ向けの URI（`tel:` ／地図スキーム）。
- **データSoR/オーナー部門**: 本アプリ内（家族共有型 Web アプリ）が SoR。オーナーは主家族メンバー（家族グループ管理者）。緊急時連絡先の編集権限は家族メンバー全員が持ち、削除は主家族メンバーに限定する想定（TBD）。
- **依存システム**: 端末標準の電話アプリ（OS 提供）、端末標準の地図アプリ（OS／ブラウザ提供）。外部 SaaS／API 連携は本 UC 内では発生しない。
- **In範囲**: 全画面常設の緊急アクション UI、登録済み連絡先のワンタップ表示・電話発信トリガー・地図起動トリガー、オフライン時のキャッシュ参照、最小監査ログ記録。
- **Out範囲**: 予約・チャット・テレヘルス、自動診断／トリアージ、家族外（獣医師本人）への通知、IoT 連動、救急車・行政窓口連携、複数言語音声ガイダンス。
- **Out理由**: 戦略オプション B のスコープ（記録・リマインド・通院前サマリ）および「診断行為を行わない」制約と、緊急時に操作工数を増やさない原則に整合させるため。
- **KPI紐づけ**: 緊急時連絡先タップから電話発信起動までの所要時間、緊急時連絡先未登録家族グループ比率（`docs/recommended-kpi-okr.md` 不在のため KPI-ID は TBD）。
- **優先度**: P0
- **未確定事項**:
  - TBD: 緊急時連絡先の登録可能件数（主＋副 2 件で十分か、夜間救急を含め 3 件まで許容するか）／影響: 休診時間帯の到達率／埋め方: 主家族メンバーへのヒアリングと `docs/business-requirement.md` L200 の確認事項解消／宛先: 主家族メンバー／区分: B
  - TBD: 全画面常設 UI の表示位置・サイズ（モバイル／PC で操作性をどう担保するか）／影響: 緊急時の発見性／埋め方: MVP UI プロトタイプでの家族レビュー／宛先: 開発者／区分: B
  - TBD: 緊急アクション監査ログの保持期間と参照 UI の有無／影響: 振り返り・改善サイクル／埋め方: MVP 後の運用要件として別途決定／宛先: 主家族メンバー／区分: NB
  - TBD: 診療時間メモから「営業中／休診中」を自動判定するロジックを MVP に含めるか／影響: 副連絡先の自動切替可否／埋め方: MVP スコープ判断（自由記述メモのみで開始し将来構造化を検討）／宛先: 開発者／区分: NB
