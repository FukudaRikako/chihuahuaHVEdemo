# UC-14 詳細 — システムが投薬・予防薬・ワクチン期日のリマインド通知を配信する

## 0. Executive Summary

- 対象提言: REC-01（記録・リマインド・通院前サマリに機能を絞った家族共有型 Web アプリの段階導入）の「リマインド」中核機能。
- 解決課題: I1（投薬・予防薬の漏れリスク・優先度高）、I5（ルナの低血糖・ワクチン管理）、I6（家族間タスク重複／抜け）の構造的解消。
- 価値: 投薬・予防薬・ワクチンの「飲ませたか分からない」事象を能動通知で撲滅し、投薬遵守率の 0 件運用化に寄与。
- 主要リスク3点と先回り策:
  1. 通知過多による通知ミュート化 → 1犬1日あたり既定上限とスヌーズ規定で抑制。
  2. 二重投薬リスク（家族複数人が同時通知受領）→ 通知に「完了済みの場合は他メンバーへ即時取消通知」連動（UC-05 と接続）。
  3. デバイス通知未許諾／オフライン → メール・アプリ内バナーへの自動フォールバック。

## 1. 前提とスコープ

- スコープ内: アプリ内通知／メール／（任意で）Web Push による期日リマインド配信。配信対象は犬ごとの投薬・月次予防薬・年次ワクチン予定。
- スコープ外: SMS／LINE 等の外部チャネル連携、診断アラート、家族外（獣医師等）への通知、IoT デバイス連携によるトリガー。
- 主要アクター: システム（スケジューラ／一次アクター）、家族メンバー（通知受信者）。
- システム境界: 内＝予定データ・通知ルール・配信キュー・配信履歴。外＝メール配信サービス、Web Push サービス、ユーザ端末。

## 2. ユースケース詳細（固定スキーマ）

- **ID**: UC-14
- **名称**: システムが投薬・予防薬・ワクチン期日のリマインド通知を配信する
- **業務エリア or ジャーニー段階**: 日次タスク遂行（投薬・予防薬・ワクチン管理）
- **目的**: 期日が到来した投薬・予防薬・ワクチンの実施を家族メンバーへ能動通知し、漏れ・遅延・二重実施を防止する。
- **提言/論点ID**: REC-01
- **一次アクター**: システム（スケジューラ）
- **二次アクター/外部システム**: 家族メンバー、メール配信サービス、Web Push サービス
- **トリガー**: スケジューラの定時起動（既定: 5 分間隔の期日評価）、または予定登録・更新イベント（UC-06／UC-13）による即時再評価。
- **前提条件**:
  - 対象犬の投薬・予防薬・ワクチン予定が UC-03／UC-06／UC-13 で登録済みであること。
  - 家族メンバーが UC-01 でサインアップ済みであり、通知チャネル設定（アプリ内／メール／Web Push）が 1 つ以上有効であること。
  - 家族グループ（UC-02）に紐づく通知配信先メンバーが解決可能であること。
- **基本フロー要約**:
  1. スケジューラが現在時刻 ± 通知前リードタイム（例: 当日朝・期日 1 時間前）に該当する未完了予定を抽出する。
  2. 予定ごとに配信対象メンバー（家族グループ内・通知 ON）と有効チャネルを解決する。
  3. 重複抑止キー（犬 ID × 予定 ID × 通知種別 × 日付）で配信履歴を照合し未送信のみ採用する。
  4. 通知ペイロード（犬名・タスク種別・期日・完了アクションリンク）を生成し、各チャネルへ送信ジョブを発行する。
  5. 配信結果（成功／失敗）を配信履歴に記録し、UC-04（今日のタスク）に未完了フラグを反映する。
- **主要代替/例外**:
  - 通知失敗（メール bounce 等）→ 別チャネルへフォールバックし配信履歴に失敗理由を記録。
  - 予定が通知前に完了記録された（UC-05／UC-06）→ 配信ジョブをキャンセルし「実施済み」として履歴に残す。
  - スケジューラ停止／実行遅延 → 復旧時に未送信ぶんを一括再評価しリードタイム超過分は「遅延通知」ラベル付きで配信。
- **事後条件**: 対象家族メンバーへ通知が 1 件以上配信され、配信履歴に犬 ID／予定 ID／チャネル／結果が記録されている。UC-04 のタスク一覧に通知済みステータスが反映される。
- **主要データI/O**:
  - 入力: 犬プロフィール、投薬予定、予防薬予定、ワクチン予定、家族メンバー、通知設定、配信履歴。
  - 出力: 通知メッセージ、配信履歴レコード、UC-04 向けタスク状態更新イベント。
- **データSoR/オーナー部門**: 本アプリ内（家族共有型 Web アプリ）が SoR。オーナーは主家族メンバー（家族グループ管理者）。
- **依存システム**: メール配信サービス（外部 SaaS、TBD）、Web Push サービス（ブラウザ標準）、アプリ内通知ストア。
- **In範囲**: 投薬・月次予防薬・年次ワクチンの期日リマインド配信、配信履歴記録、重複抑止、チャネルフォールバック。
- **Out範囲**: 診断アラート、家族外宛通知、SMS／LINE 等外部チャネル、IoT トリガー、通知文の AI 生成最適化。
- **Out理由**: 戦略オプション B のスコープ（記録・リマインド・通院前サマリ）と法・倫理制約（診断行為禁止）に整合させるため。
- **KPI紐づけ**: 投薬遵守率／予防薬遵守率／ワクチン期日遵守率（`docs/recommended-kpi-okr.md` 不在のため KPI-ID は TBD）。
- **優先度**: P0
