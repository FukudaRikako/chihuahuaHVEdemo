# UC-18: 家族メンバーが獣医師共有用 PDF サマリを出力する

- 対象事業: チワワファミリーの家庭内ペット健康管理（家族共有型 Web アプリ）
- 入力骨格: `docs/catalog/use-case-skeleton.md`（UC-18 行）
- 入力事業要件: `docs/business-requirement.md`（§推奨戦略・長期フェーズで「獣医師共有用 PDF 出力」が明示）
- 採番: `UC-18`（骨格から継承）
- 戦略オプション: B（推奨）— 専用 Web アプリ（MVP→拡張）の長期フェーズ機能

## 1. 概要

| 項目 | 内容 |
|---|---|
| ID | UC-18 |
| 名称 | 家族メンバーが獣医師共有用 PDF サマリを出力する |
| 業務エリア or ジャーニー段階 | 通院・獣医師連携（外部共有） |
| 目的 | 通院・セカンドオピニオン等の際に、犬1頭の健康経過（体重・食欲・投薬・予防薬・症状メモ）を獣医師へ共有可能な PDF として書き出し、診療精度と家族外連携品質を高める。 |
| 提言/論点ID（REC-xx） | TBD（`docs/business-requirement.md` に `REC-xx` 採番なし。関連: 長期フェーズ「獣医師共有用 PDF 出力」、Key Issue I2「通院前情報整理の高コスト」、I7「写真・症状メモの分散」） |
| 一次アクター | 家族メンバー |
| 二次アクター/外部システム | かかりつけ獣医師（PDF 受領者・アプリ外）、PDF ビューア（OS 標準）、メール/メッセージングアプリ（共有手段・アプリ外） |
| トリガー | 家族メンバーが犬詳細画面または通院前サマリ画面から「獣医師共有用 PDF 出力」を選択 |
| 前提条件 | 家族メンバーが認証済みで対象犬のアクセス権を持つ／対象犬の基本プロフィールおよび直近の健康記録が1件以上存在する／獣医師への共有許諾範囲が家族内で合意済み（TBD: 家族設定での明示確認は将来課題） |
| 基本フロー要約 | 1) 対象犬と出力期間（既定: 直近3か月）を選択。 2) 出力項目（基本プロフィール／体重推移／投薬・予防薬履歴／症状メモ・写真／通院履歴）をチェック。 3) システムが該当データを集計し PDF を生成。 4) プレビュー表示後、端末へダウンロードまたは OS 共有シート経由で送信。 5) 出力履歴を記録（誰がいつ何を出力したか）。 |
| 主要代替/例外 | E1: 対象犬に記録が0件→「出力可能なデータがありません」と表示し処理中止。 / E2: PDF 生成タイムアウト（大量写真）→ 写真を縮小再試行、それでも失敗時はエラー通知し再実行を促す。 / A1: 家族メンバーがアクセス権なし犬を選択→403 相当のエラーで拒否。 |
| 事後条件 | 指定範囲の PDF が生成され端末に保存または共有可能になっている／出力履歴が監査用に保存されている／元データは変更されない（読み取り専用処理）。 |
| 主要データI/O | 入力: 犬プロフィール、体重記録、投薬実施記録、予防薬実施記録、症状メモ、写真メタデータ、通院履歴。 出力: PDF ファイル（バイナリ）、PDF 出力履歴。 |
| データSoR/オーナー部門 | アプリ内データストア（本ソリューションが SoR）／オーナー: 家族グループ（主家族メンバー管轄） |
| 依存システム | アプリ内 PDF 生成モジュール、画像ストレージ、認証・認可サブシステム（UC-01/UC-19）、通院前サマリ生成（UC-12 と項目構成を共通化） |
| In範囲 | PDF テンプレート選択、期間・項目フィルタ、生成・プレビュー・ダウンロード・共有シート連携、出力履歴の保存。 |
| Out範囲 | 獣医師アカウントの発行・アプリ内招待、獣医師からの返信/コメント収集、自動メール送信（家族メンバーの手動共有に委ねる）、診断ロジック、電子カルテ連携 API。 |
| Out理由（短文） | 戦略オプション B のスコープ外（家族外への直接アカウント発行は禁止）／診断機能は方針上提供しない／電子カルテ連携は MVP/長期計画に未定義。 |
| KPI紐づけ | 通院前準備時間 30 分→5 分（事業要件 §推奨戦略 KPI）。サマリ生成所要時間（事業要件 §推奨戦略 KPI）。`docs/recommended-kpi-okr.md` 不在のため KPI-ID は TBD。 |
| 優先度 | P2（骨格より継承。長期フェーズ機能） |

## 2. Coverage 反映

- 主要アクター×トリガー: 家族メンバー × 「獣医師通院前/通院後の情報共有イベント」を本 UC でカバー。
- Key Issue: I2（通院前情報整理）／I7（写真・症状メモの分散）に直接寄与。
- 関連 UC: UC-12（通院前サマリ）と項目構成・集計ロジックを共通化／UC-10（統合タイムライン）・UC-11（グラフ）のデータを参照／UC-19（アクセス権限）で出力権限を制御。

## 3. Decision Log（最小）

- 2026-06-22: 獣医師共有は「家族メンバーが手動で PDF を共有」方式に限定し、獣医師アカウント発行・自動送信は Out of Scope と決定（決定者: TBD／会議体: ARD Step 4.2 自動生成時点の方針継承）。
