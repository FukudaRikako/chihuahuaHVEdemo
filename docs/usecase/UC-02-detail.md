# UC-02: 主家族が家族グループを作成し他メンバーを招待する

- 対象事業: チワワファミリーの家庭内ペット健康管理（家族共有型 Web アプリ）
- 関連戦略: 推奨オプション B（専用 Web アプリ MVP→拡張）
- 入力: `docs/business-requirement.md`、`docs/catalog/use-case-skeleton.md`

## 1. 基本情報

| 項目 | 内容 |

## 2. 基本フロー要約

1. 主家族メンバーがアプリにログインし、家族グループ未作成の場合は「家族グループを作成」画面へ誘導される。
2. グループ名・対象犬の概要（任意）を入力し家族グループを作成する。主家族メンバーがオーナーロールで自動登録される。
3. 「メンバー招待」画面で招待先メールアドレス・表示名（任意）・ロール（家族メンバー）を入力する（複数件まとめて入力可）。
4. システムが招待トークンを発行し、招待メール（参加リンク付き）を外部メール配信サービス経由で送信する。
5. 招待状態（未送信／送信済／参加済／期限切れ）が一覧で確認でき、再送・取り消し操作が可能になる（詳細運用は UC-19 で継続）。

## 3. 主要代替/例外

| # | 条件 | 結果 |
|---|---|---|
| A1 | 招待メールアドレスが既に同グループに参加済み | 重複登録を抑止し「参加済み」メッセージを表示 |
| A2 | 招待メール送信失敗（配信サービス障害／無効アドレス） | 送信失敗ステータスで保持し、原因表示と再送導線を提供 |
| A3 | 招待トークン期限切れ前に参加未完了 | 招待状態を「期限切れ」に更新し、主家族メンバーへ再招待を促す（UC-19 へ接続） |

## 4. 事後条件

- 家族グループが一意な ID で作成され、主家族メンバーがオーナーロールで所属している。
- 招待した各メンバーについて招待レコード（メールアドレス、トークン、状態、有効期限）が作成・永続化されている。
- 招待先が UC-01 のサインアップ／ログインで招待トークンを消費した時点で、家族グループのメンバー一覧に追加される。

## 5. 主要データ I/O

- 入力: グループ名、対象犬の概要（任意）、招待先メールアドレス、招待先表示名（任意）、ロール
- 出力: 家族グループ、家族グループ所属（オーナー）、招待レコード、招待メール
- 参照: ユーザーアカウント（UC-01 が作成）

## 6. データ SoR/オーナー部門

- 家族グループ／所属レコード／招待レコード: 本アプリ（家族グループサービス）。「オーナー部門」は **主家族メンバー** が代理。
- 招待メール本文・送信ログ: 外部メール配信サービス（保管期間は当該サービス仕様に依存、TBD）。

## 7. 依存システム

- 認証基盤（UC-01 と共有、具体方式は TBD）
- メール配信サービス（招待メール送信、具体製品は TBD）
- 本アプリの家族グループサービス（本 UC で構築、UC-01／UC-19 と接続）

## 8. スコープ

- In 範囲:
  - 家族グループの新規作成（1 主家族メンバー＝1 グループ前提）
  - 家族メンバーの招待（メールアドレス指定、複数件一括可）
  - 招待トークン発行・招待メール送信・状態管理（未送信／送信済／参加済／期限切れ）
  - 招待の再送・取り消し（基本操作）
- Out 範囲:
  - 家族外（祖父母・ペットシッター・獣医師本人）への招待
  - 複数グループの所有・グループ間メンバー移動
  - 招待先への SMS／電話通知
  - 詳細な権限カスタマイズ（ロール追加・粒度細分化）は UC-19 側で扱い、本 UC では扱わない
- Out 理由:
  - 家族外招待は事業要件（資料 Q3／Q6、非機能憲章「家族外共有しない」）により対象外。
  - 複数グループ所有・SMS 通知は MVP スコープ外（戦略ロードマップ §9 中長期で再評価）。
  - 権限粒度の拡張は UC-19 のスコープであり、本 UC では基本ロール（オーナー／家族メンバー）に限定する。

## 9. KPI 紐づけ

- 家族グループあたりの参加メンバー数（家族共有定着の代理指標、TBD: 目標値未設定）

## 検証

- 必須項目（ID／名称／業務エリア／目的／提言ID／一次アクター／二次アクター／トリガー／前提条件／基本フロー／代替例外／事後条件／主要I/O／SoR／依存システム／In/Out 範囲／Out 理由／KPI／優先度／未確定事項）を全て充足。
- 粒度: 「主家族が家族グループを作成し他メンバーを招待する」という一次アクターのゴール達成単位（動詞＋目的語）に準拠。画面名／単一処理名／単一 API 名のみで終わる粒度ではない。
- スコープ整合: 戦略オプション B のソフトウェア提供シナリオに限定し、家族外招待・複数グループ所有・SMS 通知・権限粒度拡張を Out of Scope に明示。UC-19 との責務分担を明確化。
- 関連 UC: UC-01（認証）に依存、UC-03〜（各種記録）の前提となる共有スコープを定義、UC-19（権限・招待状態管理）の運用フェーズへ接続。`docs/catalog/use-case-skeleton.md` の骨格と齟齬なし。
- 捏造回避: メール配信サービス・トークン有効期限・グループ所有数等の未確定事項は `TBD` として明示し、根拠のない具体製品名・数値は記載していない。
