# UC-01: 家族メンバーがアプリにサインアップ・ログインする

- 対象事業: チワワファミリーの家庭内ペット健康管理（家族共有型 Web アプリ）
- 関連戦略: 推奨オプション B（専用 Web アプリ MVP→拡張）
- 入力: `docs/business-requirement.md`、`docs/catalog/use-case-skeleton.md`

## 2. 基本フロー要約

1. 利用者がアプリ URL を開き、「サインアップ」または「ログイン」を選ぶ（招待メール経由の場合は招待リンクから直接サインアップ画面に到達）。
2. サインアップ時はメールアドレス・パスワード・表示名を入力し、招待トークンが付与されていれば紐づく家族グループに自動参加する。
3. システムが入力を検証してアカウントを作成し、必要に応じてメール確認（verify）を要求する。
4. ログイン時はメールアドレスとパスワードで認証し、認証成功後に家族グループのホーム（UC-04 今日のタスク画面）へ遷移する。
5. セッションが確立され、以降の操作（UC-03 以降）に対し家族メンバーロールでの権限が付与される。

## 3. 主要代替/例外

| # | 条件 | 結果 |
|---|---|---|
| A1 | パスワード誤り／未登録メールでのログイン試行 | エラー表示し再入力を促す。一定回数を超えた場合は一時ロック（具体回数 TBD）|
| A2 | 招待トークンの期限切れ／無効 | 招待無効メッセージを表示し、主家族メンバーへ再招待依頼を案内（UC-02／UC-19 へ誘導）|
| A3 | パスワード忘れ | 「パスワードリセット」リンクからメール経由で再設定（リセットメール送信は外部メール配信サービス依存）|

## 4. 事後条件

- 認証済みセッションが確立し、家族グループ ID と家族メンバーロールが解決済みである。
- 家族メンバーが UC-03（犬プロフィール）／UC-04（今日のタスク）／UC-05〜UC-09（各種記録）等にアクセス可能な状態である。
- 招待経由サインアップの場合、招待トークンが消費済みとなり、家族グループのメンバー一覧に当該メンバーが追加されている。

## 5. 主要データ I/O

- 入力: メールアドレス、パスワード、表示名、招待トークン（招待経由時）
- 出力: ユーザーアカウント、認証セッション（トークン）、家族メンバー所属レコード
- 参照: 家族グループ（UC-02 が作成）、招待レコード（UC-02／UC-19 が発行）

## 6. データ SoR/オーナー部門

- ユーザーアカウント／認証情報: 本アプリ（認証基盤）。本ファミリーは個人運用のため「オーナー部門」は **主家族メンバー** が代理。
- 招待レコード／家族グループ所属: 本アプリ（家族グループサービス）。

## 7. 依存システム

- 認証基盤（自前 or 外部 IdP、具体方式は TBD）
- メール配信サービス（招待・確認・リセットメール用、具体製品は TBD）
- 本アプリの家族グループサービス（UC-02 で構築）

## 8. スコープ

- In 範囲:
  - メール＋パスワードによるサインアップ／ログイン／ログアウト
  - 招待トークン経由の家族グループ参加
  - パスワードリセット（メール経由）
  - セッション確立とロール付与（家族メンバー／主家族メンバー）
- Out 範囲:
  - 家族外（祖父母・ペットシッター・獣医師本人）へのアカウント発行
  - 多要素認証（MFA）の必須化（MVP では任意機能扱い／詳細 TBD）
  - SSO・SNS ログイン（採用可否 TBD）
  - 端末生体認証連携
- Out 理由:
  - 家族外発行は事業要件（資料 Q3／Q6）により対象外。
  - MFA／SSO／生体認証は MVP スコープ外。導入是非は中長期で再評価（戦略ロードマップ §9 中期・長期）。

## 検証

- 必須項目（ID／名称／業務エリア／目的／提言ID／一次アクター／二次アクター／トリガー／前提条件／基本フロー／代替例外／事後条件／主要I/O／SoR／依存システム／In/Out 範囲／Out 理由／KPI／優先度／未確定事項）を全て充足。
- 粒度: 「家族メンバーがアプリにサインアップ・ログインする」という一次アクターのゴール達成単位（動詞＋目的語）に準拠。画面名／単一処理名／単一 API 名のみで終わる粒度ではない。
- スコープ整合: 戦略オプション B のソフトウェア提供シナリオに限定し、家族外アカウント発行・SSO・MFA 必須化・生体認証等を Out of Scope に明示。
