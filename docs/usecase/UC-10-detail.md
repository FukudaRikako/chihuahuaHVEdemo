# UC-10: 家族メンバーが犬ごとの統合タイムラインを閲覧する

## 1. ID / 名称 / 目的
- **ID**: UC-10
- **名称**: 家族メンバーが犬ごとの統合タイムラインを閲覧する
- **目的（価値）**: 投薬・予防薬・体重・食事/排泄/体調・症状メモ・写真・通院予定など、異なる種別の記録を **犬ごとに 1 本の時系列ビュー** に統合表示し、家族の誰が・いつ・何を行い・どんな変化があったかを即座に把握できる状態にする。Key Issue I2（通院前情報整理の手間）と I7（写真・症状メモの分散）を直接解消し、UC-12 通院前サマリ生成の基盤ビューとなる。

## 2. 主アクター
- **一次アクター**: 家族メンバー（閲覧者）
- **二次アクター**: システム（タイムライン集約／写真ストレージ）／他の家族メンバー（記録投入元）

## 3. 前提条件
- 家族メンバーがサインイン済み（UC-01 完了）
- 対象犬プロフィールが登録済み（UC-03 完了）
- 家族グループに所属し、対象犬への閲覧権限を持つ（UC-02 / UC-19）
- 表示対象となる記録（UC-05〜UC-09, UC-13, UC-15, UC-16 等）が 1 件以上存在する（0 件時は空状態を表示）

## 4. 基本フロー要約
1. 家族メンバーがアプリで「タイムライン」を開き、対象犬を選択する（家庭内 4 頭から 1 頭、または「全頭」）。
2. システムが既定期間（例: 直近 30 日）について、犬 ID に紐づく各種イベント（Medication / Preventive / Weight / Meal / Symptom / Visit 等）を集約・時系列降順で取得する。
3. システムが種別アイコン・記録者・実施時刻・要点（投薬名／体重値／症状カテゴリ等）・サムネイル写真を 1 行ずつ表示する。
4. 家族メンバーが種別フィルタ（投薬／体重／症状／予定 等）や期間（7日／30日／カスタム）を切替えると、システムが再集約して即時更新する。
5. 家族メンバーが個別行をタップすると、当該記録の詳細（写真原寸、メモ全文、編集元 UC への導線）を表示する。
6. 家族メンバーが画面下部までスクロールすると、追加期間分を遅延読込する。

## 5. 主要例外
- **E1: 閲覧権限なし（招待中／対象犬への権限欠如）** → 当該犬を一覧から除外し、選択時はアクセス不可メッセージを表示。
- **E2: 集約タイムアウト／部分取得失敗** → 取得済みイベントのみ表示し、未取得カテゴリを「再読込」リンク付きで明示（部分縮退）。
- **E3: 対象期間に記録 0 件** → 空状態 UI と「記録を追加」導線（UC-05/07/09 等）を表示。

## 6. 主要データ I/O
- **入力**:
  - 対象犬 ID（Dog）／「全頭」指定
  - 期間（既定 30 日／7 日／カスタム範囲）
  - 種別フィルタ（複数可）
  - 閲覧者 ID（権限判定用）
- **出力 / 表示**:
  - TimelineEntry のリスト: イベント種別、発生日時、記録者、要点テキスト、サムネイル参照、元レコード ID / 元 UC
  - ページング用カーソル（次ページ取得用）
- **連携元（読み取り）**:
  - UC-05（投薬実施）／UC-06（月次予防薬）／UC-07（体重）／UC-08（食事・排泄・体調）／UC-09（症状メモ・写真）／UC-13（通院・ワクチン予定）／UC-15（シニア犬観察）／UC-16（低血糖予防給餌）
  - PhotoAsset（UC-09 で生成された写真のサムネイル）
- **連携先**:
  - UC-12（通院前サマリ生成の入力ビュー）
  - UC-21（過去記録検索からの遷移先）
