# UC-09: 家族メンバーが症状メモと写真を添付して記録する

## 1. ID / 名称 / 目的
- **ID**: UC-09
- **名称**: 家族メンバーが症状メモと写真を添付して記録する
- **目的（価値）**: 4匹のチワワに発生した気になる症状（皮膚異常、嘔吐、跛行、目やに等）をテキストメモと写真でその場で記録し、家族間で即時共有・後日の通院前サマリに活用できる状態にする。紙ノート・複数スマホメモへの分散（Key Issue I7）を解消し、犬軸の時系列に統合する。

## 2. 主アクター
- **一次アクター**: 家族メンバー（記録者）
- **二次アクター**: 他の家族メンバー（タイムライン閲覧）／システム（写真ストレージ）

## 3. 前提条件
- 家族メンバーがサインイン済み（UC-01 完了）
- 対象犬プロフィールが登録済み（UC-03 完了）
- 家族グループに所属しており、当該犬への記録権限を持つ（UC-02 / UC-19）
- スマホまたは PC のカメラ／ファイル選択により写真を取得可能

## 4. 基本フロー要約
1. 家族メンバーが「症状を記録」を起動し、対象犬を選択する。
2. 発生日時（既定: 現在時刻）、症状カテゴリ（任意タグ: 皮膚／消化器／歩行／目鼻／その他）、自由記述メモを入力する。
3. 写真を 0〜N 枚添付する（カメラ撮影またはギャラリー選択）。
4. システムが写真を写真ストレージへアップロードし、サムネイル化する。
5. 家族メンバーが「保存」を実行する。
6. システムが犬軸の時系列イベントとして保存し、タイムライン（UC-10）に反映する。

## 5. 主要例外
- **E1: 写真アップロード失敗（通信断/容量超過）** → メモのみ先行保存し、写真は再試行キューに保留。家族メンバーに再試行 UI を提示。
- **E2: 写真サイズ／枚数上限超過** → 上限（TBD: 1枚あたり最大MB／1記録あたり最大枚数は要確認）を超えた場合、リサイズまたはエラーメッセージを表示。
- **E3: 必須項目（対象犬）未選択** → 保存をブロックし、入力を促す。
- **E4: 権限不足（招待中／閲覧専用ロール）** → 保存ボタンを非活性化し、権限の説明を表示。

## 6. 主要データ I/O
- **入力**:
  - 対象犬 ID（Dog）
  - 発生日時
  - 症状カテゴリタグ（複数可）
  - 自由記述メモ（テキスト）
  - 写真ファイル（0〜N 枚）
- **出力 / 永続化**:
  - 健康イベント（SymptomNote）レコード: 犬 ID、記録者 ID、発生日時、カテゴリ、メモ、写真参照（PhotoAsset の ID リスト）、作成日時
  - PhotoAsset: 写真ストレージ上のオブジェクト参照、サムネイル URL、サイズ、MIME
- **連携先**:
  - UC-10（統合タイムライン表示）
  - UC-12 / UC-18（通院前サマリ・獣医師共有 PDF への引用）
  - UC-21（過去記録検索）
