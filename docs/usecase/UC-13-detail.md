# UC-13 詳細 — 家族メンバーが通院・ワクチン予定を登録し管理する

- 親骨格: `docs/catalog/use-case-skeleton.md` UC-13
- 入力: `docs/business-requirement.md`（As-Is §4 / Key Issues §5 / To-Be §6 / Strategic Recommendations §9 / Roadmap §10）
- スコープ: 推奨戦略「オプション B（家族共有型 Web アプリ MVP→拡張）」。診断行為・家族外共有・紙運用継続は対象外。

## ユースケース定義

- **ID**: UC-13
- **名称**: 家族メンバーが通院・ワクチン予定を登録し管理する
- **業務エリア or ジャーニー段階**: 都度業務（通院・年次ワクチン等の予約管理／スケジュール統合）
- **目的**: 4 匹それぞれの通院予約と年次ワクチン等の予定を家族共有の単一台帳に集約し、予定忘れ・重複手配・連絡漏れを防ぐ。
- **提言/論点ID（REC-xx）**: REC-01（家族共有型 Web アプリによる記録・リマインド・通院前サマリ、`docs/business-requirement.md` §9 推奨戦略「オプション B」）
- **一次アクター**: 家族メンバー（予約登録者）
- **二次アクター/外部システム**: システム（スケジューラ：UC-14 連携）、他家族メンバー（同期受信側）、通院前サマリ生成（UC-12 連携）
- **トリガー**: 家族メンバーが動物病院から次回診察・ワクチン接種の予約を取った直後、または既存予定の変更・キャンセル発生時。
- **前提条件**:
  - UC-01 でログイン済み、かつ UC-02 で家族グループに所属している。
  - UC-03 で対象犬（4 匹のいずれか）のプロフィールが登録されている。
  - 予定種別（通院／ワクチン／その他）を選択できる区分が定義済みである。
- **基本フロー要約**:
  1. 家族メンバーが対象犬を選び「通院・ワクチン予定の追加」を開く。
  2. 種別（通院／ワクチン／その他）・日時・場所（動物病院名）・目的メモを入力する。
  3. 予定を保存する。
  4. システムは予定を家族グループへ同期し、犬ごとの統合タイムライン（UC-10）と通院前サマリ生成対象（UC-12）に組み込む。
  5. システムは事前リマインド（既定: 前日／当日）を UC-14 にスケジュールする。
- **主要代替/例外**:
  - 予定を変更・キャンセル → 該当予定を編集／取消し、家族同期と UC-14 リマインドを再スケジュール。
  - 同一犬・同一日時に既存予定あり → 重複確認ダイアログを表示し、置換・併存・中止を選択。
  - オフラインで保存不可 → ローカル下書き保持、オンライン復帰時に同期。
- **事後条件**:
  - 対象犬の通院・ワクチン予定が家族グループ全員に同期されている。
  - UC-14 が当該予定をリマインド対象として認識している。
  - UC-12 通院前サマリの「次回通院日」項目に反映されている。
- **主要データI/O（データオブジェクト名のみ）**:
  - 入力: 犬プロフィール、予定種別、日時、場所（動物病院名）、目的メモ、登録者。
  - 出力: 通院・ワクチン予定レコード、家族向け同期イベント、リマインドスケジュール。
- **データSoR/オーナー部門**: 本アプリ（家族共有型 Web アプリ）が SoR。オーナーは主家族メンバー（家族グループ管理者）。
- **依存システム**: 認証基盤（UC-01）、家族グループ管理（UC-02/UC-19）、スケジューラ（UC-14）、統合タイムライン（UC-10）、通院前サマリ生成（UC-12）、緊急時連絡先（UC-17：動物病院情報の参照）。
- **In範囲**: 通院・年次ワクチン・その他予定の登録／編集／取消し、家族同期、UC-14 への事前リマインド連携、UC-12／UC-10 への予定情報連携。
- **Out範囲**: 動物病院予約システムとの外部 API 連携、ワクチン接種の医学的妥当性判定・接種間隔の自動診断、獣医師アカウントによる直接登録、外部カレンダー（Google／iCloud 等）双方向同期、ECサイト・保険連携。
- **Out理由（短文）**: 診断行為を含めない方針（`docs/business-requirement.md` §9）、家族外（獣医師本人）への直接アカウント発行は対象外（骨格 Out of Scope）、外部カレンダー／EC／IoT 連携は戦略スコープ外。
- **KPI紐づけ**: ワクチン・通院予定の遵守（I5「ルナの低血糖・ワクチン管理」緩和、`docs/business-requirement.md` §5/§9）、通院前サマリ生成所要時間短縮（次回予定の即時参照に寄与）。KPI/OKR ID は `docs/recommended-kpi-okr.md` 未生成のため TBD。
- **優先度**: P1（骨格定義どおり。I5 を緩和するが、年次／不定期発生のため日次運用の P0 中核ログ（UC-05/06/14）に次ぐ位置付け、`docs/business-requirement.md` §10 中期スコープ）。
- **未確定事項**:
  - TBD: 年次ワクチンの正確な周期と次回予定日基準（接種日+1年か接種証明書記載日か）／影響: 次回予定算出と UC-14 リマインドのズレ／埋め方: 家族代表＋かかりつけ動物病院へ確認／宛先: 主家族メンバー／B。

  - TBD: 同一日時の重複予定の取り扱い（置換／併存／中止）方針／影響: 予定台帳の整合性と UC-10 タイムライン表示／埋め方: UX レビューで確定／宛先: 開発者＋家族代表／NB。
