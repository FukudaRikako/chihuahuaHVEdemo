# UC-04 詳細: 家族メンバーが今日のタスク一覧を確認する

- 対象事業: チワワファミリーの家庭内ペット健康管理（家族共有型 Web アプリ）
- 入力: `docs/business-requirement.md`、`docs/catalog/use-case-skeleton.md`
- 骨格参照: UC-04（主アクター: 家族メンバー / 優先度: P0）
- 関連課題 ID: I1 投薬漏れ / I5 ルナの予防管理 / I6 家族間タスク重複

## 1. 基本情報

- **ID**: UC-04
- **名称**: 家族メンバーが今日のタスク一覧を確認する
- **業務エリア or ジャーニー段階**: 日次運用（朝・夕の家事ルーチン段階）
- **目的**: 当日に犬ごとに実施すべき投薬・予防薬・給餌・観察・通院などのタスクを、家族メンバーが一覧で把握し、抜け漏れと重複対応を防ぐ。
- **提言/論点ID**: REC-01（家族共有型 Web アプリ MVP 導入）、関連: I1 / I5 / I6
- **優先度**: P0（投薬・予防薬の抜け漏れ防止に直結する中核 UX）

## 2. アクターと境界

- **一次アクター**: 家族メンバー（本ファミリーの登録ユーザー）
- **二次アクター/外部システム**:
  - システム（スケジューラ）: タスク生成・期日判定（UC-14 と連携）
  - 他の家族メンバー: 同一タスクの実施状態を共有
- **トリガー**: 家族メンバーがアプリを開く、またはホーム画面で「今日のタスク」を選択する。
- **前提条件**:
  - UC-01 でログイン済みであること。
  - UC-02 で家族グループに所属していること。
  - UC-03 で対象犬プロフィールが 1 件以上登録されていること。
  - 投薬・予防薬・通院などのスケジュール元データが登録済みであること（UC-05/06/13 等で設定）。

## 3. フロー

### 基本フロー要約（3〜5 行）

1. 家族メンバーがログイン後、ホーム画面で「今日のタスク」を開く。
2. システムが当日の日付と家族グループ ID を基に、犬ごとの未完了タスク（投薬・予防薬・給餌・観察・通院）を取得して一覧表示する。
3. 各タスクには犬名・カテゴリ・予定時間帯・担当（未割当可）・実施状態（未/済）が表示される。
4. 家族メンバーは一覧をスクロール／フィルタ（犬別・カテゴリ別）して当日対応を把握する。
5. 一覧から個別タスクを選択することで、記録 UC（UC-05/06/07/08/16 等）へ遷移できる。

### 主要代替/例外（最大 3 件）

- 対象タスクなし → 「本日のタスクはありません」プレースホルダを表示し、最新の完了履歴へのリンクを提示する。
- ネットワーク切断 → 直近キャッシュを「最終同期 hh:mm」付きで表示し、同期再試行ボタンを提示する。
- 他メンバーが直前に「実施済み」へ更新 → リアルタイム（または画面再表示時）に状態を反映し、二重実施を防ぐ通知バナーを表示する。

### 事後条件

- 家族メンバーが当日の未完了タスクを把握した状態となる。
- 一覧表示自体はデータを更新しないが、タスク選択により後続記録 UC へ正しく遷移できる。

## 4. データ・依存

- **主要データ I/O**:
  - 入力: タスクスケジュール、犬プロフィール、家族グループ、実施記録（最新状態）
  - 出力: 当日タスク一覧ビュー（犬 × カテゴリ × 予定時刻 × 状態）
- **データ SoR/オーナー部門**:
  - タスクスケジュール / 実施記録 / 犬プロフィール / 家族グループ: いずれも本アプリ（家族共有型 Web アプリ）が SoR。オーナーは本ファミリー（主家族メンバー）。
- **依存システム**:
  - 認証基盤（UC-01）
  - 通知スケジューラ（UC-14）: 同一スケジュール源を共有
  - 記録系 UC: UC-05（投薬）/ UC-06（予防薬）/ UC-07（体重）/ UC-08（食事・排泄・観察）/ UC-16（ルナ給餌）

## 5. スコープ

- **In 範囲**:
  - 当日（ローカルタイムゾーン基準）の未完了・完了タスクの一覧表示。
  - 犬別・カテゴリ別フィルタ、実施記録 UC への遷移導線。
  - 家族メンバー間で同一の一覧を共有閲覧。
- **Out 範囲**:
  - タスク内容そのものの記録・更新（各記録 UC が担当）。
  - 翌日以降の予定編集（UC-13 等が担当）。
  - 家族外（獣医師・シッター）への一覧共有（戦略制約により対象外）。
  - 自動通知配信（UC-14 が担当）。
- **Out 理由**: スコープ分離により UC 粒度を一次アクターの「当日把握」というゴール単位に保つため。
