# UC-12 詳細 — 家族メンバーが通院前サマリを自動生成・出力する

- 入力骨格: `docs/catalog/use-case-skeleton.md` の UC-12 行
- 一次根拠: `docs/business-requirement.md`（I2 通院前情報整理の高コスト、§6 To-Be、§9 推奨戦略オプション B、ロードマップ中期）
- 対象スコープ: 戦略オプション B（専用 Web アプリ MVP→拡張）配下のソフトウェア提供シナリオに限定。診断行為は対象外。

## 1. 基本情報

| 基本フロー要約（3〜5行） | 1) 家族メンバーが対象犬と対象期間（既定: 直近30日、TBD: 変更可否）を選択し「サマリ生成」を実行。2) システムが体重推移・投薬実施履歴・予防薬実施履歴・症状/写真メモ・次回予定を統合タイムラインから抽出。3) 既定テンプレートに沿ったサマリを画面表示する。4) 家族メンバーはサマリを閲覧・確認し、出力形式（テキスト/簡易ファイル）を選択して出力する。5) 出力結果を家族メンバー自身が獣医師に提示・送付する。 |

| 事後条件 | (1) 家族メンバーが対象犬のサマリを閲覧・出力済みである、(2) 生成元の記録（体重・投薬等）は変更されない、(3) 生成履歴の保持要否は TBD（要確認） |
