# UC-15 詳細 — 家族メンバーがシニア犬向け観察テンプレート（心雑音等）に沿って記録する

- 対象事業: チワワファミリーの家庭内ペット健康管理（家族共有型 Web アプリ MVP）
- 関連スケルトン: `docs/catalog/use-case-skeleton.md` UC-15
- 入力: `docs/business-requirement.md`（Key Issue I4 シニア犬ちょこの心雑音モニタ不足、To-Be 中期/長期ロードマップのシニア犬向け観察テンプレ）
- 戦略整合: オプション B（専用 Web アプリ MVP→拡張）の長期ロードマップ機能。診断行為・家族外共有・IoT 計測は対象外。

## ユースケース詳細

| 項目 | 内容 |
|---|---|

| 提言/論点ID（REC-xx） | I4（シニア犬ちょこの心雑音モニタ不足）に対応する戦略提言。REC 番号は `docs/business-requirement.md` Strategic Recommendations 準拠（未確定の場合は TBD）。長期ロードマップ「シニア犬向け観察テンプレ」項目（業務要件 §To-Be ロードマップ）に整合。 |

| 二次アクター/外部システム | 同じ家族グループの他メンバー（タイムライン UC-10 / 通院前サマリ UC-12 / 獣医共有 PDF UC-18 経由で間接利用）。獣医師は UC-18 PDF 経由の間接利用のみ。 |
| トリガー | (a) 家族メンバーが日次/週次のシニア犬観察を実施した直後、(b) UC-14 のリマインド通知（シニア観察スケジュール）受信時、のいずれか。 |
| 前提条件 | UC-01 でログイン済み。UC-02 で家族グループ所属済み。UC-03 で対象犬プロフィール（ちょこ等）が登録され、かつ「シニア観察テンプレート対象」フラグまたは年齢条件（既定: 8 歳以上、TBD）を満たす。 |
| 基本フロー要約（3〜5行） | 1) 家族メンバーが対象犬（ちょこ等）の「シニア観察」を開き、定型テンプレート（呼吸数/分、咳の有無と回数、運動後の息切れ有無、活動量、食欲、心雑音関連の主観観察 等）を表示する。2) 各項目を選択式（正常/軽度/中度/重度 等）または数値入力する。3) 自由メモ・写真（任意、UC-09 連携）を添付する。4) 保存するとタイムライン（UC-10）と通院前サマリ（UC-12）/獣医共有 PDF（UC-18）の素材ストアに反映され、家族全員から既記録として可視化される。 |
| 主要代替/例外（最大3件、各1行で「条件→結果」） | A1) 観察項目の一部が未測定（例: 呼吸数を測れていない）→該当項目のみ「未測定」で保存し、必須項目は別途警告。A2) 過去 N 回と比較して悪化傾向（例: 咳の頻度上昇）を検知→保存時に「変化あり」フラグを付与しタイムラインで強調表示（診断は行わない）。E1) 対象犬がテンプレート対象外（年齢条件未充足かつ手動フラグなし）→一般観察 UC-08 への導線を提示し本 UC は起動しない。 |
| 事後条件 | シニア観察レコードがタイムライン（UC-10）と検索インデックス（UC-21）に追加される。通院前サマリ（UC-12）および獣医共有 PDF（UC-18）の素材として集計対象になる。「変化あり」フラグが付いた場合は犬別タイムラインで視覚的に強調される。 |

| データSoR/オーナー部門 | SoR: 本 Web アプリの SeniorObservationRecord ストア。オーナー: 主家族メンバー（家族グループ単位）。テンプレート定義のオーナーも主家族メンバー（初期値はアプリ既定）。 |
| 依存システム | 認証/家族グループ（UC-01/02）、犬プロフィール（UC-03、年齢/シニアフラグ）、写真記録（UC-09）、タイムライン（UC-10）、通院前サマリ（UC-12）、リマインド通知（UC-14）、獣医共有 PDF（UC-18）、検索（UC-21）。 |
| In範囲 | シニア犬向け定型観察テンプレートに沿った日次/週次記録、項目別の選択式/数値入力、過去比較に基づく「変化あり」フラグ付与、家族メンバー間の即時共有。 |

## セルフチェック

- Out of Scope の混入なし（診断・重症度判定・IoT 計測・AI 最適化・家族外直接共有を Out 範囲に明記）。
- 粒度: 一次アクター（家族メンバー）のゴール達成単位「シニア犬向け観察テンプレに沿って記録する」で統一。画面/API 名止まりではない。
- 重複なし: 一般的な体調観察は UC-08、症状＋写真は UC-09、体重は UC-07、ルナ低血糖プロトコルは UC-16 と明確に分離。
- 抜け漏れ: 主要トリガー（日次/週次観察、リマインド受信時）をカバー。
- 依存・I/O: 認証・犬プロフィール（シニアフラグ）・タイムライン・通院前サマリ・獣医共有 PDF・通知・検索を明記。
- 未確定事項: 3 件を理由/影響/埋め方/宛先/B|NB 形式で記載（TBD1 は B = ブロッカーとして識別）。
