# Use Case Skeleton — 4匹のチワワ健康管理

- 対象事業: チワワファミリーの家庭内ペット健康管理（家族共有型 Web アプリ）
- 入力: `docs/business-requirement.md`（As-Is/To-Be/Gap/Strategic Recommendations）
- 採番ルール: `UC-<連番:00>`。Step 4.2 で per-UC 詳細生成（fan-out）の対象となる骨格のみを列挙する。
- 対象スコープ: 推奨戦略「オプション B（専用 Web アプリ MVP→拡張）」が提供するソフトウェアシナリオに限定。診断行為・家族外共有・紙運用継続は対象外。
- 関連課題 ID（参考）: I1 投薬漏れ / I2 通院前情報整理 / I3 体調変化検知 / I4 シニア犬モニタ / I5 ルナの予防管理 / I6 家族間タスク重複 / I7 写真・症状分散 / I8 アクセス制御不在。

## Out of Scope（混入防止のため明記）

- 医療診断・症状診断ロジックの提供（資料準拠の制約）。
- 家族外（祖父母・ペットシッター・獣医師本人）への直接アカウント発行。獣医師連携は UC-18 の PDF 出力に限定。
- 紙ノート・汎用カレンダー継続運用のままでの統合（戦略オプション A/D は対象外）。
- ECサイト連携（フード・薬剤購入）、保険連携、デバイス（スマート体重計等）IoT 連携。
- 家族メンバーの体調管理など犬以外の対象。

## 検証

- 件数: 21 件（指定範囲 10〜30 件、上限 50 件以内）。
- ID 形式: 全件 `UC-NN`（2桁連番）で採番済み。
- 必須列: `UC-*` / 名称 / 主アクター / 優先度 をすべて充足。
- スコープ: 戦略オプション B（推奨）のソフトウェア提供シナリオに限定し、診断・家族外共有・非ソフト施策は Out of Scope に明記。
- 課題カバレッジ: I1（UC-05/06/14）、I2（UC-10/12/18）、I3（UC-07/11）、I4（UC-15）、I5（UC-06/14/16）、I6（UC-04/05/19）、I7（UC-09/10/21）、I8（UC-01/02/19）。主要 Key Issue を全て少なくとも 1 UC でカバー。
